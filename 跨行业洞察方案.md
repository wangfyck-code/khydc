# 跨行业洞察(Cross-Industry Insight)

## TL;DR

把产品需求抽象成「母需求」,去其它成熟行业捞可迁移的成分/形态/模式/概念,用本地资料库
+ opencli 实时数据交叉验证,产出带三维评分(价值 × 可行性 × 差异化)的「可迁移机会清单」。
已固化为 Claude Code Skill `cross-industry-insight`,**两步走:先跑 skill 出洞察 → 再把结构化结果 web 化**。

> 配套 Skill:`~/.claude/skills/cross-industry-insight/SKILL.md`
> 适用:宠物食品产品企划 / 跨行业产品创新

---

## 一、要解决什么问题

做一款新产品(例:**体态管理狗粮**)时,常见的困惑是:

- 该向哪些**别的行业**借鉴?
- 借鉴时具体**关注哪些东西**?
- 怎么判断哪个借鉴点**值得做**?

本方案给出一套可复用的方法,并把它固化成一个 Claude Code Skill,**最终目标是输出一个
web/app 形式的跨行业洞察工具**。实施分两步走:

1. **第一步(已完成):** 把方法论固化成 Skill,跑一次产出洞察报告 + 结构化数据。
2. **第二步(待做):** 把结构化数据(`insight.json`)渲染成 web / dashboard。

## 二、核心方法论:反向迁移

**一句话:** 你给狗做的事,人类世界或别的行业大概率早就做过、验证过了。跨行业洞察就是
系统性地去这些"成熟方案池"里,捞可迁移的成分、形态、模式、概念。

### 第 1 步 · 把需求抽象成「母需求」

去掉物种和品类限定词,保留根本诉求。抽象得越干净,可借鉴的行业越多。

> `体态管理狗粮` → `哺乳动物的体重/体脂管理`、`把日常饮食变成管理手段`、`健康/外观焦虑的自我管理`

### 第 2 步 · 扫描对标行业(每个写明"偷什么")

以体态管理狗粮为例:

| # | 对标行业 | 偷什么 |
|---|---|---|
| 1 | 人类体重管理(代餐/轻食/控卡) | 成分原理 + 产品形态 |
| 2 | 运动营养 / 健身食品 | 高蛋白低脂配方逻辑、"增肌减脂"分型 |
| 3 | 功能性食品 / 保健品 | 饱腹技术(膳食纤维/魔芋)、益生菌调代谢、功能宣称话术 |
| 4 | 慢病饮食管理(糖尿病/低 GI) | 科学背书、低 GI 概念、控糖逻辑 |
| 5 | 减肥经济新趋势(GLP-1/司美格鲁肽) | 市场势能、用户教育、"管理而非节食"心智 |
| 6 | 数字健康 / 智能硬件 | 订阅制、按体重数据个性化、软硬结合 |
| 7 | 母婴配方奶粉 | 分阶段精准营养、产品分层 |

### 第 3 步 · 每个对标行业按「十维度」拆解

这就是"该关注哪些东西"的固定清单:

1. **需求/痛点** —— 用户为什么买?背后的情绪/焦虑?
2. **目标人群** —— 谁在买?画像(什么主人 + 什么狗/猫)。
3. **成分与技术** —— 有效成分、原理;哪些能迁移、哪些是禁区。
4. **产品形态** —— 形态创新能否迁移?
5. **概念与话术** —— 怎么讲故事、教育市场、起概念名。
6. **价格带与定价策略** —— 主流价位带、定价逻辑。
7. **商业模式** —— 订阅?个性化定制?软硬结合?
8. **竞品/头部玩家** —— 头部品牌、爆品及打法。
9. **监管/红线** —— 功效宣称的边界与雷区。
10. **数据/证据** —— 有无临床、实测或权威背书。

## 三、从洞察到决策:可迁移机会评分

把所有维度的发现,汇总成「**可迁移机会清单**」,每条机会打三个分(1–5):

- **价值 value** —— 对目标产品的潜在价值。
- **可行性 feasibility** —— 迁移落地难度(成分可得性、合规、成本、技术)。
- **差异化 differentiation** —— 是否有壁垒、是否易被竞品照抄(越高=越稀缺/越难抄)。

> 综合分 = 价值 × 可行性 × 差异化,排序选 Top 3。
> **价值高但差异化低** → 单列为「红海机会」提醒:能做,但易被抄,慎重。

## 四、数据来源(三层,本地优先)

洞察不靠拍脑袋,关键结论要用真实数据校验。来源分三层:

| 层级 | 来源 | 用途 |
|---|---|---|
| **第 1 层(优先)** | 本地宠物行业资料库:`~/Downloads/乖宝/`、`~/Downloads/乖宝迭代设计/`、本 Vault | 价格带、监管红线(犬猫粮国标)、竞品现状、市场势能、数据证据 —— 权威、私有、免采集 |
| **第 2 层** | `opencli` 实时采集(144 个数据源:淘宝/京东/小红书/抖音/PubMed/1688/雪球/亚马逊…) | 实时热销、海外趋势、最新种草、学术成分 |
| **第 3 层** | 联网搜索(WebSearch / WebFetch) | 前两层都没有时兜底 |

> 注:本地资料含销售数据与未上市规划,洞察产物若需外发,**先脱敏**。

## 五、Skill 工作流总览

调用方式(新会话内):`/cross-industry-insight 体态管理狗粮`

| 阶段 | 做什么 |
|---|---|
| 0 锁定母需求 | 抽象成 1–3 个母需求,与用户确认 |
| 1 扫描对标行业 | 列 5–8 个行业 + 各偷什么(宠物概念优先用"种子库") |
| 2 十维度拆解 | 每个行业按十维度拆,关键维度引用本地资料库 |
| 3 真实数据验证 | 对 2–4 个关键结论分层取数校验 |
| 4 迁移性评估 | 三维评分,排出 Top 3 + 红海提醒 |
| 5 输出 | 产出三份产物(见下) |

**三份产物**(写到 `~/AI/cross-industry-insight/runs/<概念>-<日期>/`):

1. **onepager.md** —— 给老板/团队看的一页纸(母需求 + Top3 机会 + 红海提醒)。
2. **report.md** —— 完整洞察报告。
3. **insight.json** —— 结构化数据,**供第二步 web 直接渲染,不必重跑分析**。

## 六、关键设计决策记录

本方案在打磨中确定了以下取舍,便于日后回看:

- **维度从 7 扩到 10** —— 补了「目标人群 / 价格带 / 竞品现状」,贴合产品企划日常决策。
- **评分从 2 维到 3 维** —— 加「差异化」维,避免被导向红海。
- **产物加「一页纸」** —— 详细报告之外,单出一张可直接对上汇报的 onepager。
- **定位:宠物食品优先 + 通用回退** —— 内置"宠物食品对标行业种子库"跑得稳;遇非宠物概念则纯动态生成。
- **数据:本地资料库优先** —— 把乖宝/麦富迪内部资料设为第 1 层,公开采集退为第 2 层。

## 七、下一步:web 化

衔接设计的关键是 `insight.json`——第一步已把分析结果结构化,第二步只需读它来渲染,**不必重跑分析**。

建议的 web 形态(待定,可沿用熟悉的 Streamlit):

- **母需求** → 顶部一句话。
- **对标行业** → 行业卡片,点开看十维度。
- **机会清单** → 价值 × 可行性散点图,点大小=差异化,红海机会标红。
- **Top 3** → 置顶推荐卡。
